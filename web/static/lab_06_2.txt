1. собрать в СРТ топологию по рисунку lab_06_2.png


Вопрос:
	=> Смогут ли РС0 и РС1 обмениваться пакетами?
	
=> Как маршрутизатор Router0 должен поступать с ARP-запросами?


2. настроить узлам сети и интерфейсам маршрутизатора адреса, указанные на рисунке lab_06_2.png


3. перейти в режим симуляции, в фильтрах выключить все протоколы, кроме ARP и ICMP


4. на узле РС0 просмотреть содержимое ARP-таблицы.


5. запустить проверку прохождения ICMP пакетов до узла РС1.


6. в режиме симуляции по шагам просмотреть процесс разрешения MAC-адреса узла РС1.


7. В режиме реального времени запустить проверку прохождения ICMP пакетов до интерфейсов Fa0/0 и Fa0/1 маршрутизатора Router0.


8. на узле РС0 просмотреть содержимое ARP-таблицы.


Вопрос:
	=> какие МАС-адреса соответствуют IP-адресам РС1 и интерфейсам Fa0/0 и Fa0/1 маршрутизатора Router0?

	=> Как это можно объяснить?


9. на маршрутизаторе выполнить:
show ip interface fastEthernet 0/0 
show ip interface fastEthernet 0/1


Вопрос:	
=> Есть ли в выводе команд информация, касающаяся ARP-протокола?

	=> Как подобное поведение маршрутизатора по отношению к ARP трафику может быть изменено?


10. Выполнить на интерфейсе fastEthernet 0/0 команду:
no ip proxy-arp
